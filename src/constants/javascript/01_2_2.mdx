## OOP의 주요 개념  
<Card className="mb-6">  
  <CardHeader>  
    <CardTitle>1. 클래스와 인스턴스</CardTitle>  
    <CardDescription>  
      클래스는 객체의 설계도이며, 인스턴스는 클래스를 기반으로 생성된 구체적인 객체입니다.  
    </CardDescription>  
  </CardHeader>  
  <CardContent>  
    ```pseudocode  
    class Professor  
        properties  
            name  
            teaches  
        methods  
            grade(paper)  
            introduceSelf()  

    walsh = new Professor("Walsh", "Psychology")  
    walsh.introduceSelf() // "My name is Professor Walsh and I will be your Psychology professor."  
    ```  
    위 예시에서 `Professor`는 클래스이며, `walsh`는 그 클래스의 인스턴스입니다.  
  </CardContent>  
</Card>  

<Card className="mb-6">  
  <CardHeader>  
    <CardTitle>2. 상속</CardTitle>  
    <CardDescription>  
      공통 속성과 메서드를 상위 클래스에서 정의하고, 이를 하위 클래스에서 재사용하거나 확장할 수 있습니다.  
    </CardDescription>  
  </CardHeader>  
  <CardContent>  
    ```pseudocode  
    class Person  
        properties  
            name  
        methods  
            introduceSelf()  

    class Professor : extends Person  
        properties  
            teaches  
        methods  
            grade(paper)  
            introduceSelf()  

    class Student : extends Person  
        properties  
            year  
        methods  
            introduceSelf()  

    walsh = new Professor("Walsh", "Psychology")  
    walsh.introduceSelf() // "My name is Professor Walsh and I will be your Psychology professor."  
    ```  
    `Person` 클래스는 `Professor`와 `Student`의 공통 요소를 정의하며, 각 하위 클래스는 자신만의 속성과 메서드를 추가할 수 있습니다.  
  </CardContent>  
</Card>  

<Card className="mb-6">  
  <CardHeader>  
    <CardTitle>3. 캡슐화</CardTitle>  
    <CardDescription>  
      객체의 내부 상태를 외부에서 직접 접근하지 못하도록 보호하고, 메서드를 통해서만 접근할 수 있게 합니다.  
    </CardDescription>  
  </CardHeader>  
  <CardContent>  
    ```pseudocode  
    class Student  
        private properties  
            year  
        methods  
            canStudyArchery() { return this.year > 1 }  

    student = new Student("Alice", 2)  
    student.canStudyArchery() // true  
    ```  
    내부 구현이 외부로부터 보호되므로, 로직 변경 시 객체 내부만 수정하면 됩니다.  
  </CardContent>  
</Card>  

## 자바스크립트에서의 OOP  
<Card className="mb-6">  
  <CardHeader>  
    <CardTitle>자바스크립트와 클래스 기반 OOP의 차이</CardTitle>  
    <CardDescription>  
      자바스크립트는 프로토타입 체인을 활용한 독특한 OOP 구현 방식을 가집니다.  
    </CardDescription>  
  </CardHeader>  
  <CardContent>  
    - 자바스크립트에서는 클래스를 사용하지 않고도 객체를 생성할 수 있습니다.  
    - 프로토타입 체인을 통해 상속이 구현되며, 이는 클래스 기반 언어의 상속과는 다른 방식으로 작동합니다.  
    - 클래스와 생성자의 구분 없이 함수와 객체 리터럴을 활용하여 더 가벼운 방식으로 객체를 생성할 수 있습니다.  

    ```javascript  
    function Person(name) {  
      this.name = name;  
    }  

    Person.prototype.introduceSelf = function () {  
      console.log(`My name is ${this.name}`);  
    };  

    const walsh = new Person("Walsh");  
    walsh.introduceSelf(); // "My name is Walsh"  
    ```  
  </CardContent>  
</Card>
